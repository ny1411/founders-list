# Design Guidelines

## Philosophy

This project should feel like a premium developer tool rather than a marketing website.

Think:

- Linear
- Vercel
- Raycast
- Stripe Docs
- Notion
- GitHub

The interface should be calm, understated, and information-first.

---

# Design Principles

## Less UI

Every element should have a purpose.

Avoid unnecessary:

- icons
- colors
- shadows
- gradients
- decorations

Whitespace creates hierarchy.

---

## Typography > Decoration

Hierarchy should come from typography, not effects.

Use:

- elegant serif for major hero headings
- clean sans-serif for everything else

Avoid:

- outlined text
- gradient text
- glowing text

---

## Borders over Shadows

Cards should feel separated by borders rather than elevation.

Good:

- 1px borders
- subtle surface colors

Avoid:

- large drop shadows
- floating cards

---

## Consistent Spacing

Use a spacing scale.

Example:

- 4
- 8
- 12
- 16
- 24
- 32
- 48
- 64
- 80

Every section should have generous breathing room.

Never cram components together.

# Typography

## Display

Large serif.

Examples:

- Instrument Serif
- Canela
- Editorial
- Noe Display

Use only for:

- Hero title
- Landing page headlines

---

## UI

Sans-serif.

Examples:

- Inter
- Geist

Use for:

- body
- navigation
- buttons
- cards
- labels

---

# Layout

Keep content left aligned.

Use a large centered container.

Structure:

Hero

↓

Description

↓

Statistics

↓

Search

↓

Content Grid

↓

Footer

Maintain generous spacing between sections.

---

# Cards

Cards should be:

- flat
- rounded
- bordered
- minimal

No shadows.

Padding:

24px

Border radius:

12–16px

Hover:

- slightly lighter background
- optional arrow movement

---

# Search

Large search field.

Contains:

- search icon
- placeholder
- keyboard shortcut hint (/)

Focus state:

- brighter border
- no glow

---

# Buttons

Simple.

Rounded.

Minimal.

Hover:

- subtle background shift

No gradients.

No shadows.

---

# Icons

Use outline icons.

Small.

Muted.

Never oversized.

---

# Animations

Duration:

150–200ms

Easing:

ease-out

Animations should be subtle.

Examples:

- hover background
- border color
- opacity
- translateY(-1px)
- arrow slide

Avoid:

- bounce
- scaling
- flashy motion

---

# Lists & Tables

Prefer:

bordered layouts

instead of

floating cards.

Dense information should remain readable.

---

# Visual Hierarchy

Hierarchy should come from:

1. Typography
2. Spacing
3. Alignment
4. Borders

Not from:

- color
- shadows
- effects

---

# Tone

The UI should feel:

- premium
- calm
- modern
- editorial
- developer-focused
- minimal
- trustworthy
- timeless

Never playful.

Never noisy.

Never flashy.

If unsure, remove visual elements instead of adding more.
